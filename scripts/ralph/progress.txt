## 2026-01-19 - US-000 (Create NumberInput Component)
- **What was implemented:**
  - Created `src/components/ui/number-input.tsx` - a reusable controlled number input
  - Component uses local string state while editing to prevent controlled input issues
  - Syncs with external value only when input is not focused
  - Zero values are correctly stored as 0, not undefined/null
  - Empty input stores as null for optional fields or shows error for required
  - Allows intermediate typing states without validation errors
  - Validates on blur with toast notifications
  - Supports min/max/step/precision, prefix/suffix display
  - Handles paste with currency symbol stripping ($, €, £, etc.)
  - Arrow key increments/decrements
  - Uses inputMode='decimal' for mobile keyboard

- **Files changed:**
  - `src/components/ui/number-input.tsx` (new file)

- **Learnings for future iterations:**
  - The pattern `parseFloat(value) || undefined` causes zero to become undefined - this is the root issue the NumberInput solves
  - Use local string state for number inputs to prevent React controlled input weirdness
  - Only sync external value when input loses focus, not on every keystroke
  - Intermediate typing states like "0.", "-", "." should not trigger validation
