## Codebase Patterns
- NumberInput component handles zero values correctly by using local string state during editing
- Use `inputMode="decimal"` for mobile-friendly number keyboards
- Toast notifications for validation errors on blur, not during typing
- Allow intermediate typing states ("0.", "-", ".") without showing errors
- Unit tests for pure utilities: use vitest.unit.config.ts (no database needed), run with `bun run test:unit`
- **Image Upload Pattern**: `use-image-upload.ts` hook provides S3 upload with progress tracking - reuse for cover images
- **Storage Router**: `storage.ts` has generic `getImageUploadUrl` endpoint - extend for strategy-specific cover images
- **Debounced Mutation**: `use-debounced-mutation.ts` provides debounce + optimistic updates, but simpler pattern in journal-editor may be better for auto-save
- **Save Status UI**: Journal editor uses `SaveStatus` type ('idle'|'saving'|'saved'|'error') with icons - reuse this pattern
- **Local State for Editing**: `EditableField` component uses local string state + sync on blur - same pattern as NumberInput
- **Marketplace Constants**: Use `src/lib/constants/marketplace.ts` for STRATEGY_CATEGORIES, TRADEABLE_INSTRUMENTS, MARKETPLACE_SORT_OPTIONS, and COVER_IMAGE_* constraints

## 2026-01-19 - US-005 (Audit Existing Utilities for Strategy Redesign)
- **What was audited:**

  ### 1. use-image-upload.ts (src/hooks/use-image-upload.ts)
  - **Can reuse for cover images**: YES
  - **Features**: Uses storage.getImageUploadUrl, shows progress with toast, returns public URL
  - **Key pattern**: Uses `XMLHttpRequest` for upload progress tracking (10-95% range)
  - **For cover images**: Can reuse directly with `context: "strategy-covers"`

  ### 2. storage.ts (src/server/api/routers/storage.ts)
  - **Has getImageUploadUrl endpoint**: YES
  - **Key pattern**: Generic endpoint that takes context, generates unique S3 key, returns presignedUrl + publicUrl
  - **S3 key format**: `images/{userId}/{context}/{uuid}-{filename}`
  - **For cover images**: Could use existing endpoint OR create strategy-specific one with size/type validation
  - **Note**: Uses nanoid for UUID, getPresignedUploadUrl from lib/storage/s3

  ### 3. use-debounced-mutation.ts (src/hooks/use-debounced-mutation.ts)
  - **Can reuse for auto-save**: PARTIALLY
  - **Features**: Debounces mutations, supports optimistic updates, per-item debouncing with getKey
  - **Exports**: `useDebouncedMutation` and `useOptimisticState` hooks
  - **For auto-save**: Simpler pattern in journal-editor (setTimeout + clearTimeout) may be better - auto-save needs conflict detection which this doesn't have

  ### 4. journal-editor.tsx (src/components/daily-journal/journal-editor.tsx)
  - **Save status pattern**: YES - excellent reference
  - **SaveStatus type**: `'idle' | 'saving' | 'saved' | 'error'`
  - **Key patterns**:
    - Uses `lastSavedContentRef` to compare before saving (prevents unnecessary saves)
    - Debounce with setTimeout/clearTimeout (500ms delay)
    - Save indicator shows: Loader2Icon (saving), CheckCircleIcon (saved in text-profit), AlertCircleIcon (error in text-loss)
    - "Saved" status auto-clears after 2 seconds
  - **For strategy auto-save**: Need to add 'conflict' status and conflict resolution UI

  ### 5. editable-field.tsx (src/components/trade-detail/editable-field.tsx)
  - **Local state pattern**: YES - same as NumberInput
  - **Key patterns**:
    - Local state (`localValue`) with `isFocused` flag
    - Sync with external value only when not focused (useEffect)
    - Track initial value to detect changes (`initialValueRef`)
    - Call onChange on blur only if changed
    - Enter to confirm, Escape to cancel (resets to initial)
    - Currency symbol stripping on paste for number fields
  - **For strategy fields**: NumberInput already uses this pattern; EditableField provides text/select variants

## 2026-01-19 - US-014 (Add Strategy Auto-Save Endpoint with Conflict Detection)
- **What was implemented:**
  - Added `autosaveStrategySchema` with clientUpdatedAt for conflict detection
  - Added `autosave` mutation to strategies router
  - Compares clientUpdatedAt (ISO timestamp) with server's updatedAt
  - If client older: returns { success: false, conflict: true, serverVersion }
  - If equal or newer: proceeds with update and returns { success: true, savedAt, strategy }
  - Handles rules update like regular update mutation

- **Files changed:**
  - `src/server/api/routers/strategies.ts` (added autosaveStrategySchema and autosave mutation)

- **Learnings for future iterations:**
  - Optimistic concurrency control pattern: compare timestamps, return conflict info for frontend resolution
  - Return serverVersion on conflict so frontend can show diff
  - Auto-save mutations should NOT invalidate queries - they're silent background operations
