## Codebase Patterns
- Bug report enums in schema: bugSeverityEnum (low, medium, high, critical), bugCategoryEnum (ui, data, performance, crash, other), bugReportStatusEnum (open, in_progress, resolved, closed)
- User role enum: userRoleEnum (user, admin)
- Constants barrel export from src/lib/constants/index.ts using `export * from "./file"`
- Biome formatter enforces single-line for short assignments - don't break lines unnecessarily
- Admin endpoints: use `adminProcedure` from `@/server/api/trpc` (extends protectedProcedure, checks role === 'admin')
- Middleware chaining: to add middleware that depends on a prior middleware's ctx additions, use `procedure.use()` inline (not standalone t.middleware)
- Admin router lives at src/server/api/routers/admin/index.ts — uses nested createTRPCRouter for sub-routers (e.g., admin.analytics.platformStats)
- For aggregating tokens from aiMessages, use `sql<string>\`coalesce(sum(...), 0)\`` since sum can return null
- Don't use `.default({})` on Zod objects with required fields that have field-level `.default()` — TypeScript rejects it
- Drizzle relational queries `findMany({ with: { user: true } })` auto-join via relations — no manual join needed
- Use correlated SQL subqueries for computed columns in list views: `sql<number>\`cast((select count(*) from "table" where ...) as integer)\``
- `ilike` from drizzle-orm for case-insensitive search; combine with `or()` for multi-field search
- Don't import schema tables only used in raw SQL strings — Biome flags unused imports
