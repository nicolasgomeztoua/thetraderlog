## Codebase Patterns

### Analytics Utilities
- **Location:** `src/lib/analytics/` contains all shared analytics utilities
- **Key file:** `stats.ts:131` - `calculateAggregateStats()` is the main function for computing trade statistics
- **Returns:** `AggregateStats` interface with totalTrades, wins, losses, breakevens, winRate, totalPnl, avgPnl, grossProfit, grossLoss, profitFactor, avgWin, avgLoss, avgRMultiple
- **Usage:** Import from `@/lib/analytics` (barrel export in index.ts)
- **Breakeven threshold:** Use `getUserBreakevenThreshold()` from `@/server/api/helpers` to get user's BE threshold

### Chart Components (Reusable)
- **EquityCurve** (`src/components/analytics/equity-curve.tsx`) - Full AG Charts cumulative P&L with drawdown highlighting
- **MetricCard** (`src/components/analytics/metric-card.tsx`) - Terminal-styled metric display with tooltips
- **RiskGauge** (`src/components/analytics/risk-gauge.tsx`) - Circular SVG gauge for percentages (0-100)
- **Note for mini charts:** EquityCurve is heavyweight (AG Charts). For small sparklines, create lightweight SVG/canvas component

### Existing Strategy Queries
- **Location:** `src/server/api/routers/strategies.ts`
- `getStats` (line 415) - Returns basic stats for a single strategy using `calculateAggregateStats`
- `getAllStats` (line 698) - Batch stats for all strategies (win rate, P&L, profit factor)
- `getComparativeData` (line 753) - Cumulative P&L curves for charting
- `getRuleCompliance` (line 461) - Per-rule and per-trade compliance percentages

### Strategy Performance Query (Needed)
- US-008 requires `getPerformanceByStrategy` that returns recent P&L series (last 20 trades) for mini charts
- Can reuse `calculateAggregateStats` for metrics
- Need to add recent trades with cumulative P&L for sparkline data

### Rules Compliance Query (Needed)
- US-012 requires `getRulesCompliance` - similar to existing `getRuleCompliance` but with per-category rollups
- Existing query already does per-rule compliance, need to add category aggregation
