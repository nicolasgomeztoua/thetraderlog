## Codebase Patterns
- Use `@openrouter/ai-sdk-provider` (not `@ai-sdk/openai`) for OpenRouter integration with Vercel AI SDK
- Vercel AI SDK v5+ uses `stopWhen: stepCountIs(N)` instead of deprecated `maxSteps`
- Tool definitions use `tool()` with Zod `inputSchema` and `execute` function
- `generateText` returns `{ text, steps, toolCalls, toolResults, usage, totalUsage, response }`
- `onStepFinish` callback available for progress tracking during tool loops
- OpenRouter provider: `createOpenRouter({ apiKey })` then `provider('model/name')`
- Existing tools have executor functions that can be reused — just wrap with Zod schemas
- DataStore pattern (Map<string, unknown>) passes through tool context — reusable in SDK format
- All AI constants live in `src/lib/constants/ai.ts` — update MAX_TOOL_ROUNDS references
- Schema context is static (generateSchemaContext), user context is async (buildUserContext)
- Report progress tracking uses fire-and-forget DB updates via updateProgress()
- Chat tool loop: MAX_TOOL_ROUNDS_CHAT = 10, Report: MAX_TOOL_ROUNDS_REPORT = 20
- System prompt built by buildSystemPrompt() in src/lib/ai/prompts/trading-analyst.ts
- Tool executor pattern: executeTool() switch-case dispatches to individual executor functions
- Report email sent via Resend after completion — unchanged in migration

|---------|---------------|

| `fetchWithRetry()` | SDK built-in retry via `maxRetries` param |
| `MAX_TOOL_ROUNDS_CHAT = 10` | `stopWhen: stepCountIs(10)` |
| `MAX_TOOL_ROUNDS_REPORT = 20` | `stopWhen: stepCountIs(20)` |

### Vercel AI SDK Key Patterns
1. **Provider setup:** `createOpenRouter({ apiKey: env.OPENROUTER_API_KEY })` from `@openrouter/ai-sdk-provider`
2. **Model selection:** `provider('moonshotai/kimi-k2')` returns a model instance
3. **Tool definition:** `tool({ description: '...', inputSchema: z.object({...}), execute: async (args) => {...} })`
4. **Multi-step:** `generateText({ ..., tools, stopWhen: stepCountIs(10) })` — SDK handles the loop
5. **Progress tracking:** `onStepFinish({ text, toolCalls, toolResults, usage, stepType })` callback
6. **Token tracking:** `result.totalUsage` aggregates across all steps
7. **Steps access:** `result.steps` array contains all intermediate tool calls/results
8. **System prompt:** Separate `system` param (not first message)

### Package Decision
- PRD says `bun add ai @ai-sdk/openai` — but we need `@openrouter/ai-sdk-provider` for OpenRouter
- Correct install: `bun add ai @openrouter/ai-sdk-provider`
- The `@ai-sdk/openai` package targets OpenAI directly, NOT OpenRouter
