## Codebase Patterns
- Analytics chart components are self-contained (data in, chart out) — ideal for MDX wrappers with dataRef
- SessionChart, HourHeatmap, CalendarHeatmap depend on useSettingsStore() for timezone — MDX wrappers need to handle this
- AI tools registry in src/lib/ai/tools/index.ts uses switch-case dispatcher pattern
- Report mode system prompt is in REPORT_MODE_INSTRUCTIONS const in trading-analyst.ts
- Trigger.dev task uses tool-calling loop with MAX_TOOL_ROUNDS_REPORT rounds
- Email uses Resend API with raw HTML template — will replace with React Email
- Report progress stages: queued → building_context → analyzing → generating_pdf → uploading → complete
- New progress stages (post-MDX): queued → building_context → analyzing → complete (remove generating_pdf, uploading)
- Schema uses createTable() helper, aiReports has pdfUrl/pdfKey columns to replace with content/dataArtifacts
- MetricCard is inline props (no dataRef needed) — can pass through directly in MDX
