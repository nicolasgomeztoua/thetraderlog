  - `checkDailyLossCompliance(trade, dailyLossLimit, dailyTrades, accountBalance?)` - Check daily loss compliance
- Features:
  - Supports both dollar-based and percent-based limits
  - Calculates cumulative P&L from prior trades on the same day (filters out current trade)

  - Handles missing account balance for percent-based limits

- **Learnings for future iterations:**
  - Daily loss limits are typically positive numbers representing max loss allowed (e.g., $400)

  - Filter prior trades by ID, not by time (caller handles date filtering)

## 2026-01-24 - US-007 Add Concurrent Positions Checker
- What was implemented: Function to check if a trade violated max concurrent positions limit
- Files changed:
  - `src/lib/strategies/risk-compliance.ts` - Added checkConcurrentPositions function
- Function signature:
  - `checkConcurrentPositions(trade, maxConcurrentPositions, allTrades)` - Check concurrent positions compliance
- Features:
  - Counts trades that were open at the current trade's entry time
  - A trade is "open" if: started before/at entry AND (no exit OR exited after entry)
  - Returns actual count + 1 (includes current trade being opened)
  - Handles missing entry time gracefully with null passed value
- Type changes:
  - Added `exitTime?: Date | string | null` to TradeForCompliance interface
- **Learnings for future iterations:**
  - When checking concurrent positions, include the current trade being opened (+1)
  - A trade is "open" at a point in time if entryTime <= time AND (exitTime is null OR exitTime > time)
  - Always handle missing time fields gracefully with descriptive notes
