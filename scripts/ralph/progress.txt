  - `clearError: () => void` - reset error state

  - `lastSavedDataRef` tracks last saved data to skip redundant saves
  - `isInitializedRef` prevents saving on component mount
  - `onSaveRef` avoids stale closure issues with onSave callback
  - Cleanup on unmount clears any pending debounce timers
- **Learnings for future iterations:**
  - Use refs for values that shouldn't trigger re-renders (lastSavedData, debounceTimer)
  - Track initialization separately to prevent auto-save on mount
  - Use a ref for the onSave callback to avoid stale closures in useCallback deps
  - Deep comparison via JSON.stringify is sufficient for form data objects
