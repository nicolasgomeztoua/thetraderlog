## Codebase Patterns
- Prop firm rules use dollar amounts (not percentages) for maxDrawdown, dailyLossLimit, profitTarget
- DrawdownType reuses the existing enum from schemas.ts: "trailing" | "static" | "eod"
- Constants files follow pattern: types at top, exported const array at bottom, `as const` not used on mutable arrays
- All constants go in src/lib/constants/ and are re-exported from index.ts
- propFieldsSchema (shared/schemas.ts) is merged into both create and update account schemas — add new prop firm fields there
- Account types are auto-inferred from Drizzle schema — no manual type updates needed
